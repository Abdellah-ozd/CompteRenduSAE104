Contexte de la mission

La Mission 2 avait pour objectif d'exploiter la base de données KDou via une interface low code. Nous avons mené une série d’analyses pour répondre à des questions précises sur les fournisseurs, les produits, et les clients de KDou. Cela a permis de mieux comprendre les données disponibles et d'identifier des éléments clés pour l’optimisation de la gestion commerciale.

Code SQL utilisé

SELECT Fournisseur.Pays, Produit.PrixUnit, Fournisseur.Ville, Fournisseur.Societe
FROM Fournisseur
INNER JOIN Produit ON Fournisseur.NoFour = Produit.NoFour
WHERE Fournisseur.Pays IN ('Allemagne', 'Italie')
AND Produit.PrixUnit <= 50
ORDER BY Produit.PrixUnit;

2.Clients nantais

Enoncé: Question f) dénombre les clients situés à Nantes.
Interprétation:Cet indicateur permet de connaître la répartition géographique des clients pour des analyses ciblées par région.
Résultat extrait: Il y a 3 clients nantais

Code SQL utilisé

SELECT Nom, Prenom
FROM Client
WHERE Ville = "Nantes"

4. Prix moyen, minimum et maximum des produits

Enoncé:Question j) calculer la moyenne, le prix minimum et le prix maximum des produits.
Interprétation:Cet indicateur fournit une analyse de la gamme de prix des produits, utile pour les décisions de tarification.

Résultat extrait:
Prix Min Prix Max Prix Moyen
2,08 € 320,00 € 33,5361 €

Code SQL utilisé

SELECT AVG(Produit.PrixUnit) AS MoyenneDePrixUnit,
MIN(Produit.PrixUnit) AS MinDePrixUnit,
MAX(Produit.PrixUnit) AS MaxDePrixUnit
FROM Produit;

Conclusion

Cette mission a permis de structurer et d'interpréter des données essentielles pour KDou, en fournissant des informations exploitables sur les produits, les clients, et les fournisseurs. Grâce aux requêtes SQL et à l’interface low code, nous avons pu extraire des indicateurs clés qui aideront KDou à optimiser sa gestion des stocks, affiner son offre, et mieux comprendre sa clientèle. Ces résultats posent des bases solides pour des décisions stratégiques éclairées et le développement futur de l’entreprise.
